"Jive Forums admin console 管理员账户密码重置"

需求：

webcenter集成AD 之后发现Discussion控件用不了，使用weblogic账号也无法进入Jive forums的控制台，但是weblogic控制台可以登录。于是想重置Jive Forums的管理账号密码。

分析：

修改jiveuser表将管理员账户密码更新即可。

技术实现：

  • 连接到FM_DISCUSSIONS schema，查询jiveuser，发现passwordhash字段值变为了”-“字符，可能是集成AD之后这边的数据被覆盖了，按字段名推测密码应该是hash加密后的密文，于是将weblogic用户的密码通过MD5加密生成密文更新该字段，重启webcenter之后，问题解决。

附： //password: Welcome1 update jiveuser set passwordhash=’b56e0b4ea4962283bee762525c2d490f’ where username=’weblogic’

虚拟机安装solaris 11的时候弹出

Long mode not supported, rebooting Press any key to reboot........
原因是solaris11只支持64位，需要把BIOS的CPU选项里Inter Virtualization Technology和Inter VT均设置为ENABLE，

然后将虚拟机的设置如下：

Virtual Machine Setting->Processors->Virtualizatoin engine->Preferred mode=Intel VT-x/EPT or AMD-v/RVI

firefox升级18之后,autoproxy失效,goagent无法使用问题解决

之前使用的是17版本的firefox，配置好了autoproxy使用goagent能够正常代理，之后升级firefox18之后，却代理不了了。重装过firefox,重装过goagent，重装过autoproxy都无效。

试过使用IE，设置代理127.0.0.1:8087+goagent可以正常代理，因此排除goagent问题。

后来查看firefox18的版本说明，才发现由于安全性禁用了部分功能猜测可能由此导致无法代理

firefox说明：

禁用不安全内容：Firefox 可以禁用HTTPS安全网页中的不安全内容，来维护用户在互联网通信中的隐私。目前可以在about:config中启用这个特性。
打开about:config，过滤栏输入：proxy  修改如下选项

extensions.autoproxy.proxyMode   auto network.proxy.type               1 signon.autologin.proxy           true 即可解决问题。

虚拟机安装EBS R12时出现错误RW-50004 解决

一般出现这个错误多半应该是库文件导致的，查看日志得知是 需要的 unzip 版本过高，RHEL上安装的是6.0的，而R12.1.1.1只支持unzip 5.X 的，很坑爹，yum erase unzip 然后重新安装unzip 5.X rpm即可解决~~
